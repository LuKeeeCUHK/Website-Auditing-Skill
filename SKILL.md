# Website Conversion Auditor Skill 框架设计稿

## 1. Skill 基本信息

### Skill 名称

`website-conversion-auditor`

### 中文名称

官网页面 UX 与转化分析助手

### 一句话定位

当用户提供官网、落地页、产品页或活动页 URL 时，该 Skill 帮助 AI 扫描页面 DOM 与视觉元素，从布局、配色、CTA、排版、响应式和转化路径等维度输出结构化分析，并给出可落地的 AB 测试方案。

---

## 2. Skill 目标

这个 Skill 的核心目标是将“官网页面体验分析”标准化，让 AI 在面对一个网页时，不只是做泛泛点评，而是能按照固定分析框架完成以下任务：

1. 扫描页面结构与视觉元素。
2. 分析页面布局、配色、CTA、排版、响应式体验等维度的优劣。
3. 提炼页面优势项，并说明对应元素与转化价值。
4. 标注待优化问题项，并说明可能造成的负面影响。
5. 输出可落地的 AB 测试方案，要求单变量设计，并明确测试核心目标与关键指标。
6. 在需要时调用预留知识库，例如品牌知识、产品知识、页面类型标准、CTA 分类标准、AB 测试方法论等。

---

## 3. 适用场景

该 Skill 适用于以下场景：

- 面对品牌官网各页面提供优化思路
- 抓取相似品牌网站的UX优势点
- 将页面分析结果沉淀为知识库、报告或优化清单

---

## 4. 用户输入

用户通常会提供以下信息：

### 必填输入

- 页面 URL

### 可选输入

- 页面类型：例如首页、产品页、活动页、注册页、结账页。
- 业务目标：例如提升点击率、加购率、注册率、购买转化率、线索提交率。
- 重点分析维度：例如 CTA、首屏、移动端、商品卡片、导航结构。
- 品牌或产品知识库。
- 竞品页面 URL。
- 输出格式偏好：例如 Markdown、表格、JSON、报告。

### 示例输入

```text
请分析这个页面：https://uk.shokz.com/
重点看首屏、CTA、移动端体验和转化路径。
请输出页面优势、待优化问题和 3 个 AB 测试方案。
```

## 6. 标准执行流程

### Step 1：打开并扫描页面

- 打开用户提供的 URL。
- 如果页面有弹窗、cookie banner、地区选择、订阅弹窗，先记录其存在，再尽量关闭或接受必要设置，以便观察页面主体。
- 扫描页面 DOM，识别主要模块、标题层级、导航、按钮、图片、视频、商品卡片、表单、页脚等元素。

### Step 2：识别页面类型和业务目标

判断页面属于哪种类型：

- 官网首页
- 产品详情页
- 活动页
- 落地页
- 注册页
- 支持页
- 结账页

同时推断页面的主要业务目标，例如：

- 品牌认知
- 商品浏览
- 商品购买
- 活动参与
- 线索收集
- 账号注册
- 售后支持

### Step 3：分析页面布局

重点观察：

- 首屏是否清楚表达核心价值。
- 页面信息顺序是否符合用户决策路径。
- 重要内容是否被放在用户容易看到的位置。
- 模块之间是否有清晰层级。
- 是否存在信息拥挤、重复、跳跃或断层。

### Step 4：分析视觉设计

重点观察：

- 配色是否符合品牌调性。
- 主色与 CTA 色是否形成有效对比。
- 字体层级是否清楚。
- 图片是否支持产品理解。
- 留白是否帮助阅读和聚焦。
- 视觉焦点是否集中在关键转化动作上。

### Step 5：分析 CTA 按钮

识别所有重要 CTA，并判断：

- CTA 文案是否明确。
- CTA 是否足够突出。
- 主 CTA 和次 CTA 是否区分清楚。
- CTA 是否出现在关键决策节点。
- CTA 数量是否过多或过少。
- CTA 是否与当前页面目标一致。

常见 CTA 分类：

| CTA 类型 | 示例 | 目的 |
|---|---|---|
| Primary CTA | Buy Now / Shop Now / Add to Cart | 直接转化 |
| Secondary CTA | Learn More / View Details / Compare | 辅助决策 |
| Engagement CTA | Join Now / Subscribe / Sign Up | 互动或留资 |
| Utility CTA | Search / Cart / Account / Support | 工具型操作 |

### Step 6：分析文案与信息表达

重点观察：

- 标题是否清楚传达核心价值。
- 副标题是否解释用户利益点。
- 文案是否具体、可信、易理解。
- 是否有品牌背书、用户评价、数据证明。
- 是否存在过度抽象或无法行动的文案。

### Step 7：分析转化路径

判断用户从进入页面到完成目标动作的路径是否顺畅：

```text
进入页面 → 理解价值 → 浏览重点内容 → 建立信任 → 点击 CTA → 完成转化
```

重点观察：

- 用户是否知道下一步该做什么。
- 是否有足够的信任背书。
- 是否减少选择成本。
- 是否存在干扰转化的元素。
- 是否有中途流失风险点。

### Step 8：分析响应式体验

重点观察：

- 移动端首屏是否保留核心信息。
- 移动端导航是否易用。
- CTA 是否容易点击。
- 图片、标题、按钮是否适配不同屏幕。
- 是否存在内容过长、按钮过小、层级混乱等问题。

### Step 9：结合知识库判断

如有相关知识库，应结合以下内容进行判断：

- 品牌定位
- 产品线信息
- 目标用户
- 业务转化目标
- 页面类型标准
- CTA 分类标准
- AB 测试规范
- 历史优秀案例

### Step 10：输出分析报告

报告必须包含：

1. 页面基础信息。
2. 页面扫描摘要。
3. 页面优势项。
4. 待优化问题项。
5. 可落地 AB 测试方案。
6. 总结建议。

## 7. 分析维度框架

### 7.1 Layout 布局

- 首屏信息是否明确。
- 页面模块顺序是否合理。
- 信息层级是否清晰。
- 重要内容是否被弱化。
- 用户是否能自然往下浏览。

### 7.2 Visual 视觉

- 主视觉是否吸引注意。
- 配色是否强化品牌和 CTA。
- 字体大小和层级是否清楚。
- 留白是否合理。
- 图片是否支撑产品卖点。

### 7.3 CTA 转化按钮

- CTA 是否显眼。
- CTA 文案是否行动导向。
- CTA 是否与页面目标一致。
- 主次 CTA 是否清楚。
- CTA 是否出现在合理位置。

### 7.4 Content 文案

- 标题是否表达核心价值。
- 副标题是否补充利益点。
- 按钮文案是否明确。
- 是否有信任背书。
- 是否存在模糊、重复或过长文案。

### 7.5 Conversion Path 转化路径

- 用户是否清楚下一步行动。
- 页面是否降低选择成本。
- 页面是否帮助用户建立信任。
- 是否存在跳出或中断风险。
- 是否有足够的决策辅助信息。

### 7.6 Responsive 响应式

- 移动端首屏是否有效。
- 移动端导航是否清晰。
- 按钮是否易点击。
- 图片和文字是否适配。
- 是否存在移动端信息过载。

## 8. 页面优势项输出要求

页面优势项必须满足：

- 指向具体页面元素。
- 说明为什么这个元素有效。
- 说明它如何支持转化。
- 避免空泛评价，例如“页面很好看”“设计不错”。

### 输出示例

| 优势项 | 对应具体元素 | 为什么有效 | 转化价值 |
|---|---|---|---|
| 首屏 CTA 明确 | Hero 区 Buy Now 按钮 | 用户进入页面后能立刻看到购买入口 | 缩短从兴趣到购买的路径，提高点击率 |
| 产品分类清晰 | 顶部 Products 导航 | 按运动、生活方式、通讯等场景分类 | 降低用户寻找产品的成本 |

---

## 9. 待优化问题项输出要求

问题项必须满足：

- 指出具体元素或模块。
- 说明问题表现。
- 标注可能负面影响。
- 给出优先级。

### 优先级标准

| 优先级 | 定义 |
|---|---|
| P0 | 严重影响转化或阻塞核心路径 |
| P1 | 明显影响理解、点击或决策 |
| P2 | 体验可优化，但不一定直接影响核心转化 |

### 输出示例

| 问题项 | 对应具体元素 | 负面影响 | 优先级 |
|---|---|---|---|
| 首屏 CTA 与背景对比不足 | Hero 区 Buy Now 按钮 | 用户可能忽略核心购买入口 | P1 |
| 多个活动信息竞争注意力 | 顶部横幅 + Hero 轮播 | 用户不清楚当前最重要行动 | P1 |

---

## 10. AB 测试方案设计要求

AB 测试必须遵守：

- 每个测试只改变一个变量。
- 每个测试必须有明确假设。
- 每个测试必须绑定核心指标。
- 不提出无法实现或无法衡量的方案。
- 优先提出小范围、低成本、可快速验证的实验。

### AB 测试字段

| 字段 | 说明 |
|---|---|
| 测试名称 | 简洁说明测试主题 |
| 单变量 | 本次只改变什么 |
| A 版本 | 当前版本或控制组 |
| B 版本 | 实验版本 |
| 核心目标 | 希望提升什么 |
| 关键指标 | 用什么数据判断结果 |

### 输出示例

| 测试名称 | 单变量 | A 版本 | B 版本 | 核心目标 | 关键指标 |
|---|---|---|---|---|---|
| Hero CTA 文案测试 | CTA 文案 | Buy Now | Shop OpenRun Pro 2 | 提升首屏 CTA 点击意愿 | Hero CTA CTR |
| CTA 颜色对比测试 | 按钮颜色 | 当前按钮颜色 | 高对比度品牌主色 | 提升按钮可见性 | CTA CTR、加购率 |
| 信任背书位置测试 | 背书位置 | 背书位于页面中段 | 首屏 CTA 下方展示销量/奖项 | 提高首屏信任感 | 首屏点击率、页面滚动深度 |

## 13. 输出规则

Skill 执行时必须遵守以下规则：

1. 必须具体指出页面元素，不要只给泛泛建议。
2. 每个优势项必须说明“为什么有利于转化”。
3. 每个问题项必须说明“可能造成什么负面影响”。
4. AB 测试方案必须是单变量测试。
5. AB 测试必须包含测试目标、变量、A/B 版本、核心指标。
6. 不要一次性建议大改版，优先给可落地的小实验。
7. 如果无法访问页面或无法看到视觉细节，需要明确说明限制。
8. 如果页面存在移动端与桌面端差异，应分别说明。
9. 如果知识库与页面观察冲突，应优先说明冲突点，不要直接假设。
10. 输出应结构化，优先使用表格。

## 16. 后续可扩展方向

该 Skill 后续可以扩展为：

- 支持竞品页面对比。
- 支持按页面类型生成不同分析框架。
- 支持输出 JSON，方便进入数据系统。
- 支持生成 Jira/Linear 优化任务。
- 支持根据 GA4 或热力图数据修正分析结论。
- 支持结合品牌规范检查视觉和文案一致性。
- 支持将分析结果沉淀为页面知识库。

---

## 17. 总结

`website-conversion-auditor` 的核心价值，是把页面体验分析从“主观点评”变成“标准化、可复用、可验证的转化优化流程”。

它不仅能帮助 AI 分析页面有什么问题，还能进一步指出：

- 哪些元素已经在帮助转化；
- 哪些元素可能阻碍转化；
- 哪些优化可以通过 AB 测试验证；
- 哪些品牌、产品和业务知识需要沉淀到知识库中长期复用。
